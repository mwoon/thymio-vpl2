Detailed Description

Type 2 exercises give a description of a program and show the player 6 VPL programs to choose from. Players can access the VPL to try programs out for themselves using the button in the upper right corner.

This component is loaded from GameInterface which also sets its properties to define an exercise.

Property Documentation

answerList : var

This property holds a list of VPL code in string format. These serve as the answers players can choose from.


next : bool

The dialogue is advanced when next is set to true.

This is used within an exercise to explain the layout or for the characters to talk about the task.


question : var

This property holds the program description that should match the correct answer in answerList.


scoreList : var

This property holds a value for each item in answerList.

The values correspond to the score given to the player for choosing the item in answerList with the matching index.


storyList : var

This property holds the story content passed to the exercise from GameInterface.

It is not yet formatted to be displayed.


storyStack : var

This property holds the script to be played when the exercise is loaded. It is similar to storyStack in GameInterface.

When Type1Exercise has finished loading, this array is initialised using the contents from storyList.


Method Documentation

closePopup()

This function is called to close the in exercise VPL editor.


completeExercise(score)

This function closes the exercise window when an answer is given by the player and sends the result to the StoryTeller.


handleDialogue(speaker, text)

This function sets the speaker and dialogue to be shown within an exercise.


handleScene(scene)

This function is used when multiple story actions should happen at the same time. See also: handleScene function in GameInterface.


shuffleOptions()

This function shuffles the two arrays answerList and scoreList so that the indices still match between each list.

It is used to randomize the answer order.

Adapted from: https://bost.ocks.org/mike/shuffle/


updateStory()

This function is used to display the next action in the exercise explanation.

See also: updateStory in GameInterface .

When the storyStack becomes empty, the DialogueBox is hidden and deactivated.
